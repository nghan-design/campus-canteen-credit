# Campus Canteen Credit

## Project Title

Campus Canteen Credit

## Project Description

Campus Canteen Credit is a decentralized meal credit system built on Soroban and the Stellar blockchain. It enables schools to issue prepaid meal credits as custom tokens, allowing students to make cashless purchases at campus canteens while providing transparent and secure credit management.

The system simplifies campus payments, reduces cash handling, and enables institutions to distribute meal allowances efficiently through blockchain technology.

## Project Vision

The vision of Campus Canteen Credit is to create a secure, transparent, and accessible digital payment ecosystem for educational institutions. By replacing traditional cash-based transactions with blockchain-powered credits, schools can improve financial management while ensuring students have convenient access to daily meals.

## Key Features

* **Custom Meal Credits:** Schools issue Campus Credit Tokens (CCT) as digital meal credits.
* **Admin-Controlled Minting:** Authorized administrators can create and distribute credits to students.
* **Peer-to-Canteen Transfers:** Students can transfer credits when purchasing food and beverages.
* **Balance Tracking:** Users can view their remaining credit balance at any time.
* **Transparent Transactions:** Every transaction is recorded on-chain for accountability.
* **Total Supply Management:** The system tracks the total amount of credits issued.
* **Secure Authentication:** Transactions require wallet authorization for security.

## Usage Instructions

1. **Deploy Contract:** Deploy the Campus Credit smart contract on Stellar Testnet.
2. **Initialize Token:** Set the admin address, token name, and symbol.
3. **Issue Credits:** The school administrator mints credits for students.
4. **Check Balance:** Students can verify their available credits.
5. **Pay at Canteen:** Students transfer credits to canteen wallets as payment.
6. **Monitor Supply:** Administrators can track the total amount of issued credits.

## Future Scope

* **Approved Vendor Whitelist:** Restrict spending to registered campus canteens only.
* **Daily Spending Limits:** Prevent misuse of meal credits.
* **Semester Expiration:** Credits can expire automatically at the end of each semester.
* **Scholarship Distribution:** Schools can distribute meal subsidies to students in need.
* **QR Code Payments:** Enable fast checkout through QR-based transactions.
* **Analytics Dashboard:** Provide spending statistics for schools and parents.
* **Integration with Student IDs:** Connect digital credits with existing campus systems.

## Technology Stack

* Rust and Soroban SDK for smart contract development.
* Stellar blockchain for decentralized and transparent record keeping.
* Freighter Wallet for transaction signing and user authentication.
